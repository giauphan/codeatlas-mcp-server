## 2024-05-18 - Caching function map in AST Analyzer
**Learning:** In the `CodeAnalyzer` (TypeScript/PHP/Python parser), during the link resolution phase for `calls` (`analyzePhpFile`, `traverseAST`, etc.), we noticed a significant bottleneck when parsing large numbers of files where a nested O(n^2) behavior occurs because the map `functionNodesByName` mapping function names to array of graph node ids was being reconstructed from scratch inside the `analyze` function for *every file parsed*, iterating over `this.nodes.keys()` continuously which scales poorly with large codebases.
**Action:** Introduce a caching layer (`functionNodesByNameCache`) directly inside `CodeAnalyzer` which is populated once, and incrementally invalidated ONLY when new nodes of type 'function' (or starting with 'function:') are added to the AST graph. This way, subsequent files don't need to rebuild the map over all functions in the entire graph, dramatically reducing the time complexity for deep/wide call analysis.
